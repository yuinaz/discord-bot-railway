
PATCH: neuro-lite single-embed progress + duplicate fix (v1)

What you get:
- New util EmbedScribe that maintains *one* message per key per channel and EDITS it in-place.
- DuplicateSuppressor to avoid repeated identical updates.
- Cog progress_embed_solo: background task that updates the "Daily Progress" embed (one message).
- Cog phash_db_command_single: replaces duplicate pHash dump responses and edits a single embed.

How to install:
1) Extract this patch at the ROOT of your repo so paths line up with 'satpambot/...'
   (keep folder structure intact).

2) Ensure your bot loads the two cogs:
   - satpambot.bot.modules.discord_bot.cogs.progress_embed_solo
   - satpambot.bot.modules.discord_bot.cogs.phash_db_command_single
   If you have an auto-loader scanning this package, they will be picked up automatically.
   Otherwise, add to your cogs loader list.

3) Add/update these env keys (satpambot_config.local.json or Render env):
   - PROGRESS_EMBED_CHANNEL_ID = <channel id where you want the progress embed>
   - PROGRESS_EMBED_KEY = daily_progress   (optional, default as shown)
   - PROGRESS_EMBED_PIN = 1                (optional, pin on first creation)
   - NEURO_LITE_DIR = data/neuro-lite      (optional; files are created if missing)
   - NEURO_AUTO_INCREMENT = 1              (set 1 if you want progress to move based on activity)
   - NEURO_AUTO_INCREMENT_STEP = 0.25      (tune the speed; small value recommended)
   - SATPAMBOT_PHASH_DB_V1_PATH = data/phash/SATPAMBOT_PHASH_DB_V1.json  (optional)
   - PHASH_DB_SINGLE_COMMAND = 1           (enable the single non-duplicating command)
   - EMBED_SCRIBE_STATE = data/state/embed_scribe.json  (where message ids are tracked)

4) Remove/disable older commands that dump pHash DB to avoid duplicates OR just keep them
   but set PHASH_DB_SINGLE_COMMAND=1 and use the 'phash_db' text command provided here.
   This command uses an internal de-duplication and an upserted embed, so repeated calls
   will just *edit* the same message.

5) Run smoke tests:
   python -m scripts.smoke_cogs.py
   # Ensure:
   #  - progress_embed_solo loaded OK
   #  - phash_db_command_single loaded OK

Notes:
- On first run, the progress embed will be pinned (configurable). After that, the cog will
  only edit that same message. If the message is deleted, it will recreate and remember the new one.
- The 'neuro_lite_memory_fix' ensures the files exist and provides a bump_progress() helper.
  Other modules can import and call it; even without that, you can enable NEURO_AUTO_INCREMENT
  to avoid being stuck at 0.0% while you wire real signals.
- The state json lives at data/state/embed_scribe.json. Commit it if you want to persist message
  ids across restarts; otherwise it will be regenerated by scanning pins/history.

Troubleshooting:
- If you still see duplicate output, search for older phash dump commands and disable them.
- If overall stays at 0.0%, enable NEURO_AUTO_INCREMENT for now, then later wire real progress:
    from satpambot.ml.neuro_lite_memory_fix import bump_progress
    bump_progress("TK", "L1", 1.0)
  Any module can call this when certain learning events happen.
